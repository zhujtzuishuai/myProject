# Changelog

## [2.4.0] - 2026-04-18

### 优化
- 左侧配置区宽度增加到 520px，适配最长内容（悬浮框、输入框默认值）
- 配置区顺序调整：运行时输入在前，检查步骤在后，符合依赖关系
- 添加输入项按钮改为全宽，与添加步骤规格统一
- 必要性 badge 改为带边框的方形按钮样式，右侧显示 ↕ 提示可点击切换

## [2.3.0] - 2026-04-18

### 优化
- 左侧配置区宽度从 380px 增加到 460px，避免内容截断
- 配置区操作顺序调整：检查步骤在前，运行时输入在后，更符合使用直觉
- 步骤卡片"必要"checkbox 改为可点击的 badge 标签，悬浮时显示必要性说明；点击可切换必要/非必要

## [2.2.0] - 2026-04-18

### 优化
- 面板改为左右布局：左侧配置区，右侧执行+结果区，避免内容过长

## [2.1.0] - 2026-04-18

### 优化
- 检查项配置从工作区本地文件迁移到全局配置，所有仓库共享同一套检查项，无需重复配置
- 打开面板时自动检测 VSCode 当前打开目录，与已注册仓库路径做包含匹配，自动切换到对应仓库
- 自动匹配成功时显示"自动匹配"标签，无需手动切换工作区
- 支持直接打开文件夹（非工作区模式）的路径匹配
- 废弃工作区本地 `.feature-checker.json`，配置统一存储在插件全局 `global.json`

## [2.0.2] - 2026-04-18

### 修复
- ResMgr 解析器无法识别类属性写法（`ui: Record<string, string> = {`），导致所有 key 查找失败

## [2.0.1] - 2026-04-18

### 修复
- 保存配置后执行区输入框未刷新，导致运行时变量无法填写、插值失败

## [2.0.0] - 2026-04-18

### 重构（Breaking Change）
- 全面重构为 Pipeline 步骤模型，废弃以 CSV 为中心的旧配置方式
- 每个检查项由若干独立**步骤**组成，步骤类型：
  - `文件/目录存在`：路径模板支持 `{{变量}}` 插值
  - `ResMgr Key 存在`：检查指定模块中是否存在某个 key，key 支持插值
  - `CSV 字段非空`：通过过滤列定位行，检查目标列是否有值
  - `角色资源检查`：自动检查 rawChar/portraits 三条路径
- 运行时输入与步骤解耦，输入项统一在配置区定义，执行区填写
- 不同检查项在操作上完全独立，互不干扰
- 配置持久化格式升级（与旧版不兼容，需重新配置检查项）

## [1.4.2] - 2026-04-17

### 修复
- 角色资源检查补零逻辑错误：原来补到5位再加前缀0变成6位，现修正为补到4位再加前缀0，最终路径共5位

### 优化
- 配置持久化拆分为两层：
  - 全局配置（工作区列表）存在插件目录 `config/global.json`
  - 工作区内配置（功能检查项、表头配置等）存在项目根目录 `.feature-checker.json`，随仓库走

## [1.4.1] - 2026-04-17

### 优化
- "值相等性"检查类型更名为"与输入值相等"
- "值存在性"更名为"值存在"
- 检查级别为"不检查"时，检查类型显示为空（禁用），不再默认为"值存在"
- 执行检查时，若存在"与输入值相等"的列，自动弹出输入框逐一收集比较值，无需在配置区预先填写

## [1.4.0] - 2026-04-17

### 重构
- 功能检查改为动态管理：可随时新增/删除检查项，新增时只填名字
- 废弃硬编码的唤灵检查（featureRegistry/petFeature），所有检查项存储在配置文件中
- 检查类型调整：移除"资源存在性"，新增"角色资源检查"（自动检查 rawChar/portraits 三条路径，数值补零到5位）
- ResMgr 映射检查简化：只检查 key 是否存在于 ResMgr.ui，不再检查映射路径
- 每个检查项支持独立配置：CSV 开关、表头检查、运行时输入、equal 绑定

## [1.3.1] - 2026-04-17

### 修复
- 修复点击"+ 新增"无反应的 bug：VSCode webview 不支持 `prompt()`，改为在 extension 侧使用 `showInputBox` 完成工作区命名

## [1.3.0] - 2026-04-17

### 优化
- 工作区配置改为自动推导：点击"+ 新增"后选择 SVN 仓库根目录，自动填充项目根目录、ResMgr 文件路径、资源根目录
- 路径不存在时给出警告提示
- 去除手动填写工作区配置的表单，界面更简洁

## [1.2.0] - 2026-04-17

### 新增
- 多工作区支持：可在插件内新增、切换、删除工作区
- 每个工作区独立维护根目录、ResMgr 配置和功能配置
- 切换工作区后所有路径自动跟随切换

## [1.1.0] - 2026-04-17

### 修复
- 修复选择目录/文件后未回填到对应输入框的 bug（target→id 映射错误）
- 修复表头 resource 类型附加参数选择目录后未更新的 bug

### 优化
- 界面整体优化：输入框高度统一、按钮对齐、间距加大
- 表头表格列宽合理分配
- 所有路径输入框均提供文件管理器选择按钮

## [1.0.0] - 2026-04-17

### 新增
- 初始版本
- 支持配置表检查：值存在性、值相等性、资源存在性、ResMgr 映射检查
- 支持必要/非必要检查级别，结果可视化
- 唤灵检查功能
- 配置持久化到插件目录
