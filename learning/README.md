# 技术成长学习路径

> 目标：从业务开发向客户端深度技术栈发展
> 建议优先级：工具链 → 引擎层 → 图形渲染

## 目录结构

```
learning/
├── 01-toolchain/     工具链开发（当前最优先）
├── 02-engine/        引擎层开发
└── 03-graphics/      图形渲染（长期目标）
```

## 各阶段入口

| 阶段 | 目录 | 预计周期 |
|------|------|----------|
| [工具链开发](./01-toolchain/README.md) | 01-toolchain | 3-6 个月 |
| [引擎层开发](./02-engine/README.md) | 02-engine | 6-12 个月 |
| [图形渲染](./03-graphics/README.md) | 03-graphics | 12 个月以上 |

## 原则

- 每个阶段有实际产出再进入下一阶段
- 工具链方向可以边学边用于日常工作
- 图形渲染需要数学基础，不要跳过 03-graphics/math
